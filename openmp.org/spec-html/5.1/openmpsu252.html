5.6.4  Ending Task Regions

Summary After finishing the execution of an OpenMP task region, the implementation executes ompd_bp_task_end.

Format

 

 
void ompd_bp_task_end(void);
  

Description The OpenMP implementation must execute ompd_bp_task_end immediately after completion of a structured-block that is associated with a non-merged task. At the point that the implementation reaches ompd_bp_task_end, the binding for ompd_get_curr_task_handle is the task that finished execution. After execution of ompd_bp_task_end, any task_handle that was acquired for the task region is invalid and should be released.
